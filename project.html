自顶向下编程方法

对于“自顶向下编程方法”这一概念，我曾接触过几次，但一直没有搜索过其详细信息。今日的学习，可以说是正式的了解。

软件工程发展了几十年，随着调配的资源越来越多，编写的项目越来越大，程序员已经难以掌控全局。假如仍然沿用过去的朴素编程方法，那么程序员的编程时间也将像摩尔定律一样指数级增长。

而解决方法之一，就是采用自顶向下的编程模式。

我们先重温一次抽象的概念。抽象是计算机科学领域的重要思想，通过将硬件架构、程序架构分为多个层级，层级与层级之间只是调用的关系，如同黑箱，可以让项目更易于分工、管理。

而自顶向下编程方法就是抽象概念的一个实例。顶层的设计只需要考虑将任务分为多个小任务，而小任务的设计则考虑分为更多个微任务，直到不可分的基本操作。


举一个洗衣机的例子。该例子将洗衣机的工作模式分为两个层级。


第一层如下。是不是看起来很清晰易懂？

注水()

浸泡()

电机转动()

排水()

甩干()

停机()

第二层如下。第二层聚焦于小任务的实现。

注水(){
	water_in_switch(open)
	repeat{
		wait()
	}until(get_water_volume() >= 预定水面高度)
	water_in_switch(close)
}

浸泡(){
	time_counter = 0
	while(time_counter < 预定浸泡时间){
		wait()
		increment time_counter
	}
}

电机转动(){
	电机次数 = 0
	while(电机次数 < 预定电机次数){
		motor_run(left)
		wait()
		motor_run(right)
		wait()
		increment 电机次数
	}
}

排水(){
	water_out_switch(open)
	repeat{
		wait()
	}until(get_water_volume == 0)
}

甩干(){
	time_counter = 0
	while(time_counter < 预定甩干时间){
		motor_run(right)
		increment time_counter
	}
	water_out_switch(close) //On purpose
}

停机(){
	halt(SUCCESS)
}

这样的层级关系显然是更符合逻辑的，因而也更易于管理。同时，小任务可以交由不同的程序员来设计，以达到分工合作的目的。
